# GitHub Release Checklist

Use this before publishing a new version of the skill.

## Package Quality

- Run `quick_validate.py` on the skill directory.
- Run `scripts/check_environment.ps1` on a Windows machine with Visio installed.
- Render `assets/generic-architecture-example.json` to preview PNG and `.vsdx`.
- Run `scripts/audit_figure_spec.py` on every bundled example spec.
- Keep private paper images, proprietary model diagrams, and unpublished manuscript figures out of the public repository.

## Release Artifacts

Recommended GitHub release contents:

- Source archive generated by GitHub.
- A short release note with:
  - New capability summary.
  - Compatibility notes: Windows + Microsoft Visio for `.vsdx` rendering.
  - Dependency notes: Python, Pillow, PyYAML.
  - Known limitations.
- Optional screenshots:
  - Reference-first example preview.
  - Journal-style example preview.

## Version Note Template

```markdown
## vX.Y.Z

### Added
- Project-aware scan for paper/model directories.
- YAML-to-figure first-pass drafting for YOLO/RT-DETR-like configs.
- Journal style profiles for reference-first, Nature-like, IEEE-like, Elsevier-like, and IOP/SPIE-like figures.

### Changed
- Reference figure is now the primary style source; journal profiles only refine submission style.

### Known Limitations
- Arbitrary Python `forward()` graphs still require Codex interpretation.
- Final journal compliance requires checking the target journal's current artwork guide.
- `.vsdx` rendering requires Windows and desktop Microsoft Visio.
```
